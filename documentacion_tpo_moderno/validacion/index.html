💡 ¿Qué mide el R²?
El coeficiente de determinación indica qué porcentaje de la variabilidad de la demanda explica el modelo. Un R² de 0.90 significa que el modelo captura el 90% del comportamiento del SKU.

Umbrales usados en TPO v3.0:
• ≥ 0.85 — Modelo excelente: las predicciones son confiables para tomar decisiones de precio.
• 0.50–0.85 — Modelo aceptable: usar las recomendaciones con precaución, validar con el área comercial.
• < 0.50 — Modelo débil: el SKU tiene comportamiento muy volátil, pocos datos o es nuevo. Sus escenarios tienen alta incertidumbre.

¿Qué hacer con R² bajo?
Verificar si el SKU tiene menos de 20 semanas de historial, si tuvo cambios de surtido (des_material nuevo) o si su demanda depende más de eventos de temporada que del precio.
Barras verdes = SKUs confiables. Barras rojas = SKUs que requieren revisión manual.
¿Qué hacer con cada nivel de R²?
R² ≥ 0.85  Usar recomendación con confianza. El modelo aprendió bien la curva precio-demanda.
R² 0.50–0.85  Validar con el área comercial antes de implementar. Usar como referencia.
R² < 0.50  No recomendar promo basada en este modelo. Investigar: ¿SKU nuevo? ¿Pocos datos? ¿Outliers?

MAE (error absoluto medio): en las mismas unidades que la demanda (Kg).

💡 ¿Qué es la elasticidad precio-demanda?
Mide cuánto cambia la demanda cuando cambia el precio. Se calcula como:
ε = (ΔDemanda% / ΔPrecio%)

Interpretación:
• ε < 0 (negativa) — Comportamiento normal: al bajar el precio, la demanda sube. Esto es lo esperado para productos de gran consumo en supermercados.
• ε ≥ 0 (positiva) — Error del modelo: predijo que bajar el precio baja la demanda. Indica que el RF no aprendió correctamente la curva para ese SKU.

Valor típico en FMCG:
Elasticidades entre -0.5 y -3.0 son comunes para salsas y pastas. Valores más negativos (-5 a -10) indican SKUs muy sensibles al precio. Valores cercanos a 0 indican demanda inelástica (la promo no sirve mucho).

Nota v3.0: El filtro de elasticidad < 0 aplica solo en el reporte final (_9_). El reporte completo (_8_) mantiene todos los escenarios para análisis.
Verde = elasticidad negativa (correcto). Rojo = positiva (error del modelo). El pico verde más alto debería estar entre -2 y -0.5 para productos de gran consumo.
Valores de referencia para el negocio
ε = -0.5: SKU poco sensible al precio. Bajar 10% el precio sube 5% la demanda.
ε = -1.0: SKU elástico unitario. Bajar 10% sube 10% la demanda.
ε = -2.0: SKU muy sensible. Bajar 10% duplica el incremento de demanda (20%).
ε = -3.0: SKU altamente promocionable. Responde muy bien a descuentos.

Rango típico FMCG canal moderno: -0.8 a -2.5
Los SKUs con elasticidad entre -1.5 y -3.0 son los mejores candidatos a promo.

💡 ¿Cómo elige el optimizador la ventana ideal?
Para cada SKU y precio candidato, evalúa 98 combinaciones (7 días de inicio × 14 duraciones de 1 a 14 días). Para cada combinación calcula el ROI usando la fórmula GTN y selecciona la que maximiza el retorno.

¿Por qué importa el día de inicio?
No todos los días tienen el mismo volumen de ventas. Un SKU de pastas puede vender el 30% de su semana los sábados y solo el 8% los lunes. Poner la promo el lunes es menos eficiente que ponerla el sábado. El perfil diario ponderado por SKU×subcadena captura esto.

¿Qué indica la distribución de días de inicio?
Si todos los SKUs recomiendan el mismo día (ej. 90% el lunes), puede indicar que el perfil diario no está funcionando y el modelo está usando distribución uniforme como fallback. Una distribución variada entre días indica que el perfil real está siendo utilizado.
El día más frecuente (barra más oscura) es el óptimo de la mayoría de SKUs. Una distribución muy concentrada indica que revisar el perfil_diario.
Duraciones cortas (1-3 días) generan más tandas al mes. Duraciones largas (7-14 días) capturan más volumen por evento.
Barras verdes = ROI positivo (promo rentable). Barras rojas = ROI negativo (promo destructiva de valor).
📐 ¿Cómo se calcula el ROI en TPO v3.0?
Fórmula GTN Nueva Propuesta (feb 2026), todo en UCOs (Unidades de Consumo):

Inversión = (Precio Base − Precio Promo) × Unidades Promo
Venta Bruta = Precio Base × Unidades Promo
BI Total = Venta Bruta − Inversión − COGS
BI Unitario = BI Total / Unidades Promo
Uplift (UN) = Unidades Promo − Unidades Base
BI Uplift = BI Unitario × Uplift (UN)
ROI = (BI Uplift / Inversión) − 1

Interpretación del ROI:
• ROI ≥ 50% — Excelente: por cada S/ 100 invertidos en descuento se generan ≥ S/ 50 de beneficio incremental. Promo muy recomendada.
• ROI 0–50% — Aceptable: la promo genera retorno positivo pero moderado. Evaluar si el incremento de volumen justifica la operación.
• ROI negativo — La promo destruye valor: el descuento no se recupera con el incremento de demanda. No recomendada.

Factor de descuento: Rango permitido 6%–20% (regla dura de negocio). Descuentos fuera de rango no son operativos y se excluyen del reporte final.

💡 ¿Qué es la elasticidad precio-demanda?
Mide cuánto cambia la demanda cuando cambia el precio. Se calcula como:
ε = (ΔDemanda% / ΔPrecio%)

Interpretación:
• ε < 0 (negativa) — Comportamiento normal: al bajar el precio, la demanda sube. Esto es lo esperado para productos de gran consumo en supermercados.

Valor típico en FMCG:
Elasticidades entre -0.5 y -3.0 son comunes para salsas y pastas. Valores más negativos (-5 a -10) indican SKUs muy sensibles al precio. Valores cercanos a 0 indican demanda inelástica (la promo no sirve mucho).

💡 ¿Cómo elige el optimizador la ventana ideal?
Para cada SKU y precio candidato, evalúa 98 combinaciones (7 días de inicio × 14 duraciones de 1 a 14 días). Para cada combinación calcula el ROI usando la fórmula GTN y selecciona la que maximiza el retorno.

¿Por qué importa el día de inicio?
No todos los días tienen el mismo volumen de ventas. Un SKU de pastas puede vender el 30% de su semana los sábados y solo el 8% los lunes. Poner la promo el lunes es menos eficiente que ponerla el sábado. El perfil diario ponderado por SKU×subcadena captura esto.

¿Qué indica la distribución de días de inicio?
Si todos los SKUs recomiendan el mismo día (ej. 90% el lunes), puede indicar que el perfil diario no está funcionando y el modelo está usando distribución uniforme como fallback. Una distribución variada entre días indica que el perfil real está siendo utilizado.
El día más frecuente (barra más oscura) es el óptimo de la mayoría de SKUs. Una distribución muy concentrada indica que revisar el perfil_diario.
Duraciones cortas (1-3 días) generan más tandas al mes. Duraciones largas (7-14 días) capturan más volumen por evento.

📐 ¿Cómo se calcula el ROI en TPO v3.0?
Fórmula GTN Nueva Propuesta (feb 2026), todo en UCOs (Unidades de Consumo):

Inversión = (Precio Base − Precio Promo) × Unidades Promo
Venta Bruta = Precio Base × Unidades Promo
BI Total = Venta Bruta − Inversión − COGS
BI Unitario = BI Total / Unidades Promo
Uplift (UN) = Unidades Promo − Unidades Base
BI Uplift = BI Unitario × Uplift (UN)
ROI = (BI Uplift / Inversión) − 1

Interpretación del ROI:
• ROI ≥ 50% — Excelente: por cada S/ 100 invertidos en descuento se generan ≥ S/ 50 de beneficio incremental. Promo muy recomendada.
• ROI 0–50% — Aceptable: la promo genera retorno positivo pero moderado. Evaluar si el incremento de volumen justifica la operación.
• ROI negativo — La promo destruye valor: el descuento no se recupera con el incremento de demanda. No recomendada.

Factor de descuento: Rango permitido 6%–20% (regla dura de negocio). Descuentos fuera de rango no son operativos y se excluyen del reporte final.

💡 ¿Qué mide el R²?
El coeficiente de determinación indica qué porcentaje de la variabilidad de la demanda explica el modelo. Un R² de 0.90 significa que el modelo captura el 90% del comportamiento del SKU.

Umbrales usados en TPO v3.0:
• ≥ 0.85 — Modelo excelente: las predicciones son confiables para tomar decisiones de precio.
• 0.50–0.85 — Modelo aceptable: usar las recomendaciones con precaución, validar con el área comercial.
• < 0.50 — Modelo débil: el SKU tiene comportamiento muy volátil, pocos datos o es nuevo. Sus escenarios tienen alta incertidumbre.

¿Qué hacer con R² bajo?
Verificar si el SKU tiene menos de 20 semanas de historial, si tuvo cambios de surtido (des_material nuevo) o si su demanda depende más de eventos de temporada que del precio.
Barras verdes = SKUs confiables. Barras rojas = SKUs que requieren revisión manual.
¿Qué hacer con cada nivel de R²?
R² ≥ 0.85  Usar recomendación con confianza. El modelo aprendió bien la curva precio-demanda.
R² 0.50–0.85  Validar con el área comercial antes de implementar. Usar como referencia.
R² < 0.50  No recomendar promo basada en este modelo. Investigar: ¿SKU nuevo? ¿Pocos datos? ¿Outliers?

MAPE (error porcentual medio): mide el error de predicción en %. MAPE < 15% es bueno.
MAE (error absoluto medio): en las mismas unidades que la demanda (Kg).

💡 ¿Qué es la elasticidad precio-demanda?
Mide cuánto cambia la demanda cuando cambia el precio. Se calcula como:
ε = (ΔDemanda% / ΔPrecio%)

Interpretación:
• ε < 0 (negativa) — Comportamiento normal: al bajar el precio, la demanda sube. Esto es lo esperado para productos de gran consumo en supermercados.
• ε ≥ 0 (positiva) — Error del modelo: predijo que bajar el precio baja la demanda. Indica que el RF no aprendió correctamente la curva para ese SKU.

Valor típico en FMCG:
Elasticidades entre -0.5 y -3.0 son comunes para salsas y pastas. Valores más negativos (-5 a -10) indican SKUs muy sensibles al precio. Valores cercanos a 0 indican demanda inelástica (la promo no sirve mucho).

Nota v3.0: El filtro de elasticidad < 0 aplica solo en el reporte final (_9_). El reporte completo (_8_) mantiene todos los escenarios para análisis.
Verde = elasticidad negativa (correcto). Rojo = positiva (error del modelo). El pico verde más alto debería estar entre -2 y -0.5 para productos de gran consumo.
Valores de referencia para el negocio
ε = -0.5: SKU poco sensible al precio. Bajar 10% el precio sube 5% la demanda.
ε = -1.0: SKU elástico unitario. Bajar 10% sube 10% la demanda.
ε = -2.0: SKU muy sensible. Bajar 10% duplica el incremento de demanda (20%).
ε = -3.0: SKU altamente promocionable. Responde muy bien a descuentos.

Rango típico FMCG canal moderno: -0.8 a -2.5
Los SKUs con elasticidad entre -1.5 y -3.0 son los mejores candidatos a promo.

💡 ¿Cómo elige el optimizador la ventana ideal?
Para cada SKU y precio candidato, evalúa 98 combinaciones (7 días de inicio × 14 duraciones de 1 a 14 días). Para cada combinación calcula el ROI usando la fórmula GTN y selecciona la que maximiza el retorno.

¿Por qué importa el día de inicio?
No todos los días tienen el mismo volumen de ventas. Un SKU de pastas puede vender el 30% de su semana los sábados y solo el 8% los lunes. Poner la promo el lunes es menos eficiente que ponerla el sábado. El perfil diario ponderado por SKU×subcadena captura esto.

¿Qué indica la distribución de días de inicio?
Si todos los SKUs recomiendan el mismo día (ej. 90% el lunes), puede indicar que el perfil diario no está funcionando y el modelo está usando distribución uniforme como fallback. Una distribución variada entre días indica que el perfil real está siendo utilizado.
El día más frecuente (barra más oscura) es el óptimo de la mayoría de SKUs. Una distribución muy concentrada indica que revisar el perfil_diario.
Duraciones cortas (1-3 días) generan más tandas al mes. Duraciones largas (7-14 días) capturan más volumen por evento.
Barras verdes = ROI positivo (promo rentable). Barras rojas = ROI negativo (promo destructiva de valor).
📐 ¿Cómo se calcula el ROI en TPO v3.0?
Fórmula GTN Nueva Propuesta (feb 2026), todo en UCOs (Unidades de Consumo):

Inversión = (Precio Base − Precio Promo) × Unidades Promo
Venta Bruta = Precio Base × Unidades Promo
BI Total = Venta Bruta − Inversión − COGS
BI Unitario = BI Total / Unidades Promo
Uplift (UN) = Unidades Promo − Unidades Base
BI Uplift = BI Unitario × Uplift (UN)
ROI = (BI Uplift / Inversión) − 1

Interpretación del ROI:
• ROI ≥ 50% — Excelente: por cada S/ 100 invertidos en descuento se generan ≥ S/ 50 de beneficio incremental. Promo muy recomendada.
• ROI 0–50% — Aceptable: la promo genera retorno positivo pero moderado. Evaluar si el incremento de volumen justifica la operación.
• ROI negativo — La promo destruye valor: el descuento no se recupera con el incremento de demanda. No recomendada.

Factor de descuento: Rango permitido 6%–20% (regla dura de negocio). Descuentos fuera de rango no son operativos y se excluyen del reporte final.

💡 ¿Qué mide el R²?
El coeficiente de determinación indica qué porcentaje de la variabilidad de la demanda explica el modelo. Un R² de 0.90 significa que el modelo captura el 90% del comportamiento del SKU.

Umbrales usados en TPO v3.0:
• ≥ 0.85 — Modelo excelente: las predicciones son confiables para tomar decisiones de precio.
• 0.50–0.85 — Modelo aceptable: usar las recomendaciones con precaución, validar con el área comercial.
• < 0.50 — Modelo débil: el SKU tiene comportamiento muy volátil, pocos datos o es nuevo. Sus escenarios tienen alta incertidumbre.
