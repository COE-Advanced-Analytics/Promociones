Fórmula Financiera
★
ROI GTN — Nueva propuesta (feb 2026)
v2.0 mezclaba kilos con UCOs al calcular el beneficio incremental, produciendo ROI negativos por error de unidades. v3.0 unifica todo en UCOs para que inversión y uplift sean comparables.
Inversión = (P.Base − P.Promo) × Unidades
BI Uplift = BI Unitario × (Un.Promo − Un.Base)
ROI = (BI Uplift / Inversión) − 1
↑ Métrica financiera correcta y alineada con GTN

¿Por qué estas 5 variables?

Cada variable explica una dimensión distinta de la "oportunidad de promo". La ponderación refleja qué tan directamente impacta al ROI del optimizer.

VariablePesoPor qué importa
Elasticidad / Uplift 30% Es el driver principal del ROI. Si el consumidor no responde al precio, la promo no genera uplift y el ROI es negativo.
Margen bruto 25% Define el BI Unitario. A mayor margen, cada unidad incremental genera más utilidad. Con margen negativo, el ROI nunca es positivo.
Volumen (kg) 20% Determina el impacto absoluto. Una promo en categoría pequeña puede tener buen ROI pero irrelevancia financiera.
Señal promo histórica 15% Cuántas semanas históricas tuvieron precio promocional. Más señal = modelo entrenado con más variabilidad de precio = predicciones más confiables.
CV de precio 10% Coeficiente de variación del precio histórico. CV bajo = precio estable = curva de demanda más limpia. CV muy alto puede indicar outliers que distorsionan el RF.
⚠ Hallazgo — Categorías actuales

¿Qué significa R² = 0.989?
El modelo explica el 98.9% de la variabilidad de la demanda semanal. Es decir, de cada 100 kg de variación en ventas, el modelo predice correctamente 98.9 kg. Esto es excepcional para datos de ventas retail.
¿Por qué la elasticidad negativa solo es ~56%?
R² alto + elasticidad positiva = el modelo aprende bien la demanda pero algunos SKUs no tienen suficientes semanas de precio promocional para aprender la respuesta al precio. No es un error del modelo, es falta de señal histórica.

Paso 4 — Cálculo del ROI (Fórmula GTN)

Con ventana óptima seleccionada (Lunes, 1 día, S/ 2.45):

── INPUTS ──────────────────────────
Precio base = S/ 2.85 / kg
Precio promo = S/ 2.45 / kg
Un. promo (1d) = 5,500 × 48% = 2,640 u
BI unitario = S/ 0.82 / kg
── CÁLCULO ─────────────────────────
Inversión = (2.85 − 2.45) × 2,640 = S/ 1,056
Base 1d = 2,881 × 48% = 1,383 u
Uplift = 2,640 − 1,383 = 1,257 u
BI Uplift = 0.82 × 1,257 = S/ 1,031
── RESULTADO ───────────────────────
ROI = (1,031 / 1,056) − 1 ≈ +97%

El ROI del reporte (+194%) refleja el run completo del RF con el COGS real del sistema. Este ejemplo usa valores aproximados para ilustrar la mecánica.
